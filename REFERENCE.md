# 게임 완주 계획표 참고 문서

이 문서는 비슷한 개인용 정적 웹앱을 만들 때 참고하기 위한 설계 기록이다. `README.md`는 사용자 설명이고, 이 문서는 앱의 구조, 저장 방식, UI 방향을 전달하기 위한 개발 참고 자료다.

## 앱 목적

- 개인이 게임 목록과 예상 플레이 시간을 입력하고, 생활 리듬에 맞춰 완주 일정을 계산하는 웹앱이다.
- 서버 없이 GitHub Pages에서 실행하는 정적 웹앱이다.
- 데이터는 Dropbox의 JSON 파일 하나를 원본으로 사용한다.
- Dropbox 앱 설치나 로컬 동기화 폴더 설정 없이, Dropbox 로그인 권한으로 직접 읽고 저장한다.

## 기술 방향

- HTML, CSS, JavaScript만 사용한다.
- 프레임워크, 빌드 도구, 백엔드 서버, 데이터베이스를 쓰지 않는다.
- 앱은 `index.html`을 중심으로 동작한다.
- 화면 구조는 `index.html`, 스타일은 `styles.css`, 상태/계산/저장은 `app.js`에 둔다.
- 개인 데이터는 GitHub에 올리지 않는다.
- GitHub에는 빈 초기 JSON 템플릿만 포함한다.

## 파일 구조

- `index.html`: 앱 화면 구조
- `styles.css`: 전체 디자인과 반응형 레이아웃
- `app.js`: 상태 관리, 일정 계산, 가져오기/내보내기, Dropbox 연동
- `game-backlog-planner.json`: 빈 초기 데이터 템플릿
- `README.md`: 사용자용 설명
- `REFERENCE.md`: 비슷한 앱을 만들기 위한 참고 문서

## 데이터 저장 방식

- 기본 데이터 파일 이름은 `game-backlog-planner.json`이다.
- Dropbox 연결 후에는 Dropbox 안의 `/game-backlog-planner.json`을 원본 데이터로 사용한다.
- Dropbox 파일이 있으면 앱 시작 또는 연결 시 그 파일을 불러온다.
- Dropbox 파일이 없으면 GitHub에 포함된 빈 `game-backlog-planner.json`을 템플릿으로 새 파일을 만든다.
- Dropbox 연결 전에는 브라우저 `localStorage`를 임시 저장소로 사용한다.
- JSON 내보내기/가져오기는 수동 백업 기능으로 남겨둔다.
- CSV 내보내기/가져오기는 게임 목록 중심의 보조 기능이다.

## Dropbox 연동 방향

- Dropbox 하나만 지원한다.
- 사용자는 앱에서 `Dropbox 연결`을 누르면 Dropbox 로그인/권한 화면으로 이동한다.
- Dropbox App key는 코드에 고정해두고, 화면에 입력칸을 노출하지 않는다.
- App secret은 절대 코드에 넣지 않는다.
- OAuth PKCE 방식으로 연결한다.
- 요청 권한은 파일 읽기, 파일 쓰기, 파일 정보 읽기에 필요한 범위만 사용한다.
- 저장 시 Dropbox 파일의 `rev`를 사용해 오래된 화면이 최신 파일을 덮어쓰지 않게 한다.
- 충돌이 감지되면 사용자가 Dropbox에서 다시 불러온 뒤 수정하도록 안내한다.

## UI 방향

- 첫 화면은 실제 사용 가능한 앱 화면이어야 한다.
- 랜딩페이지, 홍보용 히어로, 긴 기능 설명 페이지를 만들지 않는다.
- 주요 입력 폼은 왼쪽, 결과 목록과 요약은 오른쪽에 둔다.
- 자주 쓰는 저장 동작은 상단에 둔다.
- 하단 백업 영역에도 Dropbox 관리 버튼을 둔다.
- 불필요한 설정 입력칸은 화면에 보이지 않게 한다.
- 버튼과 입력창은 작고 명확하게 유지한다.
- 카드 안에 카드를 중첩하지 않는다.
- 섹션은 작업 흐름 순서대로 배치한다.

## 상태 구조

앱 상태는 대략 아래 구조를 따른다.

```json
{
  "settings": {
    "startDate": "2026-07-02",
    "theme": "light"
  },
  "games": [],
  "blockedPeriods": [],
  "rules": []
}
```

게임 데이터는 세 가지 플레이 시간 기준을 가진다.

```json
{
  "id": "uuid",
  "title": "게임명",
  "platform": "Steam",
  "note": "",
  "times": {
    "mainStory": 120,
    "mainExtra": 180,
    "completionist": 300
  },
  "timeMode": "mainExtra"
}
```

## 구현 시 유지할 점

- 기존 `normalizeState` 같은 정규화 함수를 둬서 과거 JSON도 최대한 읽을 수 있게 한다.
- 저장 로직은 한 곳으로 모아두고, 사용자가 수정하는 모든 흐름에서 같은 저장 함수를 거치게 한다.
- Dropbox 연결이 되어 있으면 자동 저장하고, 연결이 없으면 localStorage에만 저장한다.
- 화면 렌더링과 데이터 저장은 분리한다.
- 계산 로직은 DOM에 직접 의존하지 않게 유지한다.
- 사용자 개인 데이터가 포함된 JSON을 커밋하지 않는다.

비슷한 앱을 만들 때는 다음처럼 요청하면 된다.

```text
이 앱과 같은 방식의 개인용 정적 웹앱을 만들고 싶어.
REFERENCE.md의 구조와 방향을 참고해줘.
서버 없이 GitHub Pages에서 실행하고, Dropbox JSON 파일 하나를 원본 데이터로 쓰게 해줘.
```
